Best Fit Facts and Dimensions - Data Modeling

Source: NN.ADM_* HR sample schema (Oracle-like) — Tables: COUNTRIES, REGIONS, LOCATIONS, DEPARTMENTS, EMPLOYEES, JOBS, JOB_HISTORY

Analysis Summary

This model presents a best-fit, Snowflake-style data warehouse design for the HR domain using the given NN.ADM_* sample schema. The design creates normalized dimension tables (snowflake-friendly) and a central fact table that captures employee history with measures (salary, commission). The model emphasizes referential integrity via surrogate keys and supports date-bounded history via a DIM_DATE dimension.

Key design choices:

  • Dimensions (dim) derived from NN.ADM_* tables: Region, Country, Location, Department, Job, Employee
  • Fact: FACT_EMPLOYEE_HISTORY capturing employee job history, salary, commission, and the associated dimension lookups
  • DIM_DATE to support START_DATE and END_DATE references from JOB_HISTORY
  • Snowflake-ready SQL blocks for creating and populating the dimensions and fact

Dimension Tables (Snowflake-ready)

Summary of each dimension, its surrogate key, and core attributes:

  • DIM_REGION: REGION_ID (PK), REGION_NAME
  • DIM_COUNTRY: COUNTRY_ID (PK), COUNTRY_NAME, REGION_ID (FK)
  • DIM_LOCATION: LOCATION_ID (PK), STREET_ADDRESS, CITY, STATE_PROVINCE, POSTAL_CODE, COUNTRY_ID (FK)
  • DIM_DEPARTMENT: DEPARTMENT_ID (PK), DEPARTMENT_NAME, MANAGER_ID, LOCATION_ID (FK)
  • DIM_JOB: JOB_ID (PK), JOB_TITLE, MIN_SALARY, MAX_SALARY
  • DIM_EMPLOYEE: EMPLOYEE_ID (PK), FIRST_NAME, LAST_NAME, EMAIL, PHONE_NUMBER, HIRE_DATE
  • DIM_DATE: DATE_KEY (PK), FULL_DATE, DAY, MONTH, YEAR, QUARTER

Fact Table

FACT_EMPLOYEE_HISTORY captures the historical facts for employee job assignments, including measurable attributes:

  • FACT_EMPLOYEE_HISTORY: EMPLOYEE_ID, JOB_ID, DEPARTMENT_ID, LOCATION_ID, START_DATE, END_DATE, SALARY, COMMISSION_PCT

Notes: START_DATE and END_DATE reference DIM_DATE via DATE_KEY values; DEPARTMENT_ID links to DIM_DEPARTMENT, which in turn links to DIM_LOCATION, and so on.

Snowflake SQL to Populate Dimensions and Facts

1) Create Dimension Tables

-- DIMENSION: Regions
CREATE OR REPLACE TABLE DIM_REGION (
  REGION_ID NUMBER PRIMARY KEY,
  REGION_NAME VARCHAR(25)
);

-- DIMENSION: Countries
CREATE OR REPLACE TABLE DIM_COUNTRY (
  COUNTRY_ID CHAR(2) PRIMARY KEY,
  COUNTRY_NAME VARCHAR(40),
  REGION_ID NUMBER,
  FOREIGN KEY (REGION_ID) REFERENCES DIM_REGION(REGION_ID)
);

-- DIMENSION: Locations
CREATE OR REPLACE TABLE DIM_LOCATION (
  LOCATION_ID NUMBER PRIMARY KEY,
  STREET_ADDRESS VARCHAR(40),
  CITY VARCHAR(30),
  STATE_PROVINCE VARCHAR(25),
  POSTAL_CODE VARCHAR(12),
  COUNTRY_ID CHAR(2),
  FOREIGN KEY (COUNTRY_ID) REFERENCES DIM_COUNTRY(COUNTRY_ID)
);

-- DIMENSION: Departments
CREATE OR REPLACE TABLE DIM_DEPARTMENT (
  DEPARTMENT_ID NUMBER PRIMARY KEY,
  DEPARTMENT_NAME VARCHAR(30),
  MANAGER_ID NUMBER,
  LOCATION_ID NUMBER,
  FOREIGN KEY (LOCATION_ID) REFERENCES DIM_LOCATION(LOCATION_ID)
);

-- DIMENSION: Jobs
CREATE OR REPLACE TABLE DIM_JOB (
  JOB_ID VARCHAR(10) PRIMARY KEY,
  JOB_TITLE VARCHAR(35),
  MIN_SALARY NUMBER(6),
  MAX_SALARY NUMBER(6)
);

-- DIMENSION: Employees
CREATE OR REPLACE TABLE DIM_EMPLOYEE (
  EMPLOYEE_ID NUMBER(6) PRIMARY KEY,
  FIRST_NAME VARCHAR(20),
  LAST_NAME VARCHAR(25),
  EMAIL VARCHAR(30),
  PHONE_NUMBER VARCHAR(20),
  HIRE_DATE DATE
);

-- DIMENSION: Date
CREATE OR REPLACE TABLE DIM_DATE (
  DATE_KEY INT PRIMARY KEY,
  FULL_DATE DATE,
  DAY INT,
  MONTH INT,
  YEAR INT,
  QUARTER INT
);

2) Create Fact Table

-- FACT: Employee History (measures + FKs to dimensions)
CREATE OR REPLACE TABLE FACT_EMPLOYEE_HISTORY (
  EMPLOYEE_ID NUMBER(6),
  JOB_ID VARCHAR(10),
  DEPARTMENT_ID NUMBER(4),
  LOCATION_ID NUMBER(4),
  START_DATE DATE,
  END_DATE DATE,
  SALARY NUMBER(8,2),
  COMMISSION_PCT NUMBER(2,2)
);

3) Populate Dimensions from Source NN.ADM_* Tables

-- Regions
INSERT INTO DIM_REGION (REGION_ID, REGION_NAME)
SELECT REGION_ID, REGION_NAME
FROM NN.ADM_REGIONS;

-- Countries
INSERT INTO DIM_COUNTRY (COUNTRY_ID, COUNTRY_NAME, REGION_ID)
SELECT COUNTRY_ID, COUNTRY_NAME, REGION_ID
FROM NN.ADM_COUNTRIES;

-- Locations
INSERT INTO DIM_LOCATION (LOCATION_ID, STREET_ADDRESS, CITY, STATE_PROVINCE, POSTAL_CODE, COUNTRY_ID)
SELECT LOCATION_ID, STREET_ADDRESS, CITY, STATE_PROVINCE, POSTAL_CODE, COUNTRY_ID
FROM NN.ADM_LOCATIONS;

-- Departments
INSERT INTO DIM_DEPARTMENT (DEPARTMENT_ID, DEPARTMENT_NAME, MANAGER_ID, LOCATION_ID)
SELECT DEPARTMENT_ID, DEPARTMENT_NAME, MANAGER_ID, LOCATION_ID
FROM NN.ADM_DEPARTMENTS;

-- Jobs
INSERT INTO DIM_JOB (JOB_ID, JOB_TITLE, MIN_SALARY, MAX_SALARY)
SELECT JOB_ID, JOB_TITLE, MIN_SALARY, MAX_SALARY
FROM NN.ADM_JOBS;

-- Employees
INSERT INTO DIM_EMPLOYEE (EMPLOYEE_ID, FIRST_NAME, LAST_NAME, EMAIL, PHONE_NUMBER, HIRE_DATE)
SELECT EMPLOYEE_ID, FIRST_NAME, LAST_NAME, EMAIL, PHONE_NUMBER, HIRE_DATE
FROM NN.ADM_EMPLOYEES;

4) Populate Date Dimension (range based on source dates)

-- Populate a broad date range (example: 2000-01-01 to 2030-12-31)
INSERT INTO DIM_DATE (DATE_KEY, FULL_DATE, DAY, MONTH, YEAR, QUARTER)
SELECT
  TO_NUMBER(TO_VARCHAR(d, 'YYYYMMDD')) AS DATE_KEY,
  d AS FULL_DATE,
  DAY(d) AS DAY,
  MONTH(d) AS MONTH,
  YEAR(d) AS YEAR,
  QUARTER(d) AS QUARTER
FROM (
  SELECT DATEADD(DAY, seq4(), '2000-01-01') AS d
  FROM TABLE(GENERATOR(ROWCOUNT => 40000)) -- ~110 years
) AS dates
WHERE d BETWEEN DATE '2000-01-01' AND DATE '2030-12-31';

5) Populate Facts from Job History (with joins to dims)

-- Fact: Employee History (link to dims via keys)
INSERT INTO FACT_EMPLOYEE_HISTORY (
  EMPLOYEE_ID, JOB_ID, DEPARTMENT_ID, LOCATION_ID, START_DATE, END_DATE, SALARY, COMMISSION_PCT
)
SELECT
  jh.EMPLOYEE_ID,
  jh.JOB_ID,
  jh.DEPARTMENT_ID,
  d.LOCATION_ID,
  jh.START_DATE,
  jh.END_DATE,
  jh.SALARY,
  jh.COMMISSION_PCT
FROM NN.ADM_JOB_HISTORY jh
LEFT JOIN NN.ADM_DEPARTMENTS d ON jh.DEPARTMENT_ID = d.DEPARTMENT_ID;

Notes: START_DATE and END_DATE are expected to map to DIM_DATE.DATE_KEY as a surrogate key for time dimensions in a full implementation. Adjust as needed if you prefer a DATE_KEY derived mapping.
